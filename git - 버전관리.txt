git add 파일이름 파일이름 파일이름 -> 해당 파일만 추가
git add . -> 현재폴더의 모든 변경점을 추가

버전 기록 : git commit -m "메시지"

어떤 버전이 기록되어 있는지 확인 : git log

되돌리기 : git reset 옵션 커밋ID

옵션 : 
--SOFT -> 커밋 내용은 사라지는데 내용을 임시 저장상태로 되돌린다.

--MIXED -> 임시 저장도 취소한다. 작업공간(local)에는 남겨둠.

--HARD -> 임시 저장도 취소, 작업공간에서도 삭제

HEAD : 최신 버전을 의미한다. 두개전으로 되돌리기는 ~2

원격저장소에 올리기 전 반드시 커밋이 완료된 상태여야한다.

     add.      commit        push
폴더 -> stage -> 버전기록 -> 원격 저장소에 저장

git (버전관리 프로그램)

버전을 만들어서 관리하는 과정
작업공간(작업) -> 스테이지(임시저장) -> 커밋(버전확정)

명령어로 관리하는 방법
1. 변경된 파일을 스테이지로 올리는법 : gut add.
2. 버전을 확정하는 방법 : git commit -m "메시지"

프로그램으로 관리하는 방법
ㄴ githubdesktop

1. 작업공간에서 작업을 하면 자동으로 스테이지에 올라간다.
2. 메시지 적고
3. push origin 버튼을 눌러야 github로 올라감
